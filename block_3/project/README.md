# Проект построения DWH и аналитического слоя

## 1. Что было сделано в проекте:
1. **Предобработка данных:** В Jupyter-ноутбуке реализовано чтение исходных данных из разных форматов (`.csv`, `.xlsx`, `.json`, `.xml`) и их базовая очистка.
2. **Проектирование архитектуры:** Создана структура базы данных **«Звезда» (Star Schema)**. Написан файл `create_star_schema.sql` с таблицами измерений (`dim_`) и фактов (`fact_`).
3. **Аналитический слой:** Написаны 5 SQL-запросов для расчета бизнес-метрик.

Важное примечание: На этапе интеграции была выявлена проблема с добавлением сырых данных в базу данных из-за обнаруженных дубликатов в идентификаторах клиентов на стороне источника. Спроектированная структура СУБД строго заблокировала импорт некорректных строк для защиты целостности таблиц. Проект успешно зафиксирован на этапе полностью готовой архитектуры, написанных запросов и выявленной аномалии в качестве данных.

## 2. Структура проекта (Что значат файлы)

```text
├── data/                           # Папка с исходными данными проекта
│   ├── customers.csv               # Данные по клиентам
│   ├── payments.csv                # Данные по платежам
│   ├── products.xlsx               # Каталог товаров (в формате Excel)
│   ├── orders.json                 # Информация о заказах (в формате JSON)
│   ├── events.xml                  # Логи активности пользователей (в формате XML)
│   ├── data_errors.csv             # Лог ошибок, найденных при чистке данных
│   └── warehouse.db                # Файл локальной базы данных SQLite
│
├── ddl/                            
│   └── create_star_schema.sql      # SQL-скрипт, который создает таблицы "Звезды" с нуля
│
├── sql/                            
│   ├── q_vip_customers.sql         # Запрос для поиска самых прибыльных клиентов
│   ├── q_monthly_sales.sql         # Запрос для ежемесячного отчета по выручке
│   ├── q_top_products.sql          # Запрос для анализа популярных товаров по категориям
│   ├── q_user_clicks.sql           # Запрос для анализа активности пользователей (клики)
│   └── q_order_statuses.sql        # Запрос для мониторинга статусов заказов для логистики
│
├── src/                            # Папка с исходным кодом проекта
│   ├── data_preprocessing.ipynb    # Jupyter-ноутбук, где происходит чтение и чистка данных
│   └── create_db.py                # Скрипт для инициализации пустой базы данных
│
├── docker-compose.yml              
├── requiremants.txt                
└── README.md                       # Этот файл с описанием проекта
